# Zotero 文献自动化纠偏与批量导入方案
## PRD v2.1 — 精简单机版（BibTeX 输入）

---

## 1. 项目背景与目标

### 1.1 痛点

在毕业论文写作中，使用 AI 辅助检索文献后，存在以下三个连续性问题：

1. DOI 偏误：AI 生成的文献元数据，尤其是 DOI、作者名、期刊名，存在幻觉或错误，直接导入会引入错误数据。
2. 批量导入困难：Zotero 的标准操作流不适合从 AI 生成的结构化结果中一键批量导入，手动逐篇处理耗时极长。
3. 问题文献无感知：若 Crossref 无法匹配某篇文献，而系统只是跳过，则最终会出现“漏引但不自知”的问题。

### 1.2 目标

本项目的目标是开发一个 Python 脚本，用于完成以下工作：

- 从 `reference.bib` 文件中读取 AI 生成的参考文献条目。
- 使用 Crossref API 对 DOI 和元数据进行自动校验与纠偏。
- 使用 Pyzotero API 自动创建 Zotero Collection 并批量导入文献。
- 对无法自动校验成功的条目，仍然导入 Zotero，但显式标记为 `⚠️ [需手动检查]`，避免静默遗漏。

### 1.3 本版本不处理的内容

以下功能不属于本次实现范围：

- 标题前加入 `[01] [02]` 序号前缀。
- 论文定稿后再批量清理标题前缀的脚本。
- Markdown 到 Word 的 Pandoc / Better BibTeX 自动联动工作流。
- 多设备同步下的排序一致性问题。

本版本面向单机使用场景，优先解决“纠偏、导入、显式标记问题条目”三件事。

---

## 2. 核心逻辑流程

- Step 1：读取并解析 `reference.bib`
- Step 2：提取文献字段并保留条目顺序
- Step 3：优先验证原始 DOI
- Step 4：若 DOI 无效，则基于标题、作者、年份进行 Crossref 模糊搜索
- Step 5：根据匹配分数决定是否接受结果
- Step 6：创建新的 Zotero Collection
- Step 7：将成功匹配的条目正常导入，将失败条目以 `⚠️ [需手动检查]` 形式降级导入
- Step 8：输出控制台汇总报告

---

## 3. 输入格式规范

### 3.1 输入文件

脚本输入文件为 `reference.bib`。

该文件由上游 AI 工具输出，用于保存本篇文章相关的参考文献。

### 3.2 选择 BibTeX 的原因

当前 pipeline 无法直接输出 JSON 格式的参考文献列表，但通常可以输出 `.bib` 文件。BibTeX 本质上也是一种结构化文本格式，每条文献由一个条目组成，字段与 JSON 十分接近，因此适合作为本脚本的输入。

BibTeX 不是 JSON，但可以稳定解析为 Python 字典列表，因此对脚本实现来说，功能上等价于结构化数据源。

### 3.3 输入字段映射

脚本从每个 BibTeX 条目中提取以下字段：

| 输入来源 | 内部字段 | 必填 | 说明 |
|---|---|---|---|
| 条目顺序 | `order` | 是 | 按 `.bib` 文件中条目出现顺序生成 |
| 标题字段 | `title` | 是 | 文献标题 |
| 作者字段 | `author` | 是 | 第一作者姓氏，供模糊搜索使用 |
| 年份字段 | `year` | 是 | 发表年份 |
| 期刊字段 | `journal` | 否 | 期刊名，作为辅助信息 |
| DOI 字段 | `doi` | 否 | 若 AI 提供则读取，否则为空 |

### 3.4 顺序前提说明

本方案默认 `.bib` 文件中的条目顺序等于文中引用的出现顺序。

如果所用 AI 工具输出 `.bib` 时是按字母顺序、相关度或其他逻辑排序，而不是按正文引用顺序排列，那么该脚本无法自动恢复原文中的真实引用顺序。

因此，在首次使用时，需要人工确认：

- `.bib` 的条目顺序是否与 Markdown 或正文中的引用顺序一致。
- 若不一致，应先手动调整 `.bib` 顺序，再执行脚本。

## 4. 技术规范

### 4.1 依赖库

需要安装以下 Python 依赖：

- `pyzotero`
- `habanero`
- `bibtexparser`

### 4.2 依赖说明

| 库名 | 用途 |
|---|---|
| `pyzotero` | 调用 Zotero Web API，创建 Collection、批量导入条目 |
| `habanero` | 调用 Crossref API，校验 DOI、搜索标准文献 |
| `bibtexparser` | 解析 `.bib` 文件，提取标题、作者、年份、DOI 等字段 |

### 4.3 配置项

建议在脚本顶部维护如下配置项：

| 配置项 | 含义 |
|---|---|
| `zotero_library_id` | Zotero 用户库 ID |
| `zotero_api_key` | 拥有 library access 和 write access 的 API Key |
| `zotero_library_type` | 个人库填 `user`，群组库填 `group` |
| `crossref_mailto` | 用于进入 Crossref Polite Pool 的邮箱 |
| `crossref_score_threshold` | 模糊匹配的最低接受分数，建议默认值为 30 |
| `input_bib` | 输入 `.bib` 文件路径 |
| `collection_prefix` | 新建 Collection 名称前缀 |

### 4.4 Zotero API 配置要求

创建 API Key 时，必须开启以下权限：

- 允许访问文库
- 允许写入文库

否则脚本无法创建 Collection，也无法导入文献。

## 5. 功能模块设计

### 5.1 `parse_refs(bib_path: str) -> List[Dict]`

#### 功能

读取 `reference.bib`，解析所有条目，并转换为统一的数据结构列表。

#### 处理逻辑

1. 使用 `bibtexparser` 读取文件。
2. 按条目在 `.bib` 中的出现顺序生成 `order`。
3. 提取 `title`、`author`、`year`、`journal`、`doi`。
4. 对缺失字段填空字符串。
5. 对标题中的花括号进行清理。
6. 从作者字段中提取第一作者姓氏，用于后续模糊搜索。

#### 输出字段

| 字段 | 说明 |
|---|---|
| `order` | 条目顺序 |
| `title` | 文献标题 |
| `author` | 第一作者姓氏 |
| `year` | 年份 |
| `journal` | 期刊名 |
| `doi` | 原始 DOI |

---

### 5.2 `validate_and_correct(ref_list: List[Dict]) -> List[Dict]`

#### 功能

对每条文献执行 DOI 校验与 Crossref 纠偏。

#### 逻辑流程

对于每条参考文献，按以下顺序处理：

1. 优先验证原始 DOI
   - 若 `doi` 非空，则调用 Crossref 对该 DOI 进行校验。
   - 若成功返回有效结果，则将该 DOI 作为 `corrected_doi`。
   - 若校验失败，则进入模糊搜索阶段。

2. 模糊搜索补全 DOI
   - 使用标题、作者、年份作为检索条件。
   - 调用 Crossref 搜索接口。
   - 取 Top 1 结果。
   - 若匹配分数大于等于阈值，则接受该结果。
   - 若匹配分数低于阈值，则标记为人工检查。

3. 生成状态字段
   - `corrected_doi`
   - `needs_review`

#### 输出新增字段

| 字段 | 类型 | 含义 |
|---|---|---|
| `corrected_doi` | 字符串或空 | 校验或搜索得到的最终 DOI |
| `needs_review` | 布尔值 | 是否需要人工检查 |

---

### 5.3 `import_to_zotero(ref_list: List[Dict]) -> None`

#### 功能

将校验后的文献列表导入 Zotero，并为本次导入创建新的 Collection。

#### 处理逻辑

1. 创建新 Collection，名称格式为 `Paper_Refs_YYYYMMDD`。
2. 遍历参考文献列表并构造 Zotero 条目。
3. 对于匹配成功的条目：
   - 按标准期刊文章类型导入。
   - 填入标准 DOI 和基础元数据。
4. 对于匹配失败的条目：
   - 不跳过。
   - 以降级方式导入。
   - 标题前加上 `⚠️ [需手动检查]`。
   - 保留 AI 原始作者、年份、期刊信息。
   - 在附加字段中说明其 DOI 校验失败。

5. 批量提交到 Zotero。
6. 输出导入汇总。

## 6. 两项核心优化

### 6.1 优化一：Crossref Polite Pool

#### 问题

Crossref 对普通请求存在速率限制。批量处理几十篇文献时，容易触发限流。

#### 方案

初始化 Crossref 客户端时加入邮箱参数，使请求进入 Polite Pool。

#### 作用

- 降低被限流的概率。
- 提高批量校验稳定性。
- 将请求间隔从 1 秒缩短到约 0.3 秒成为可行策略。

#### 推荐策略

- 设置真实邮箱。
- 每次请求后保守暂停约 0.3 秒。
- 若出现限流，再执行重试逻辑。

---

### 6.2 优化二：无 DOI 文献的 Fallback 降级导入

#### 问题

若 Crossref 搜不到结果，而脚本直接跳过该文献，会导致 Zotero 中缺少这篇文献，后续插引文时才会暴露问题。

#### 方案

即使 Crossref 校验失败，也仍然导入 Zotero，只是以“待人工核查”的形式导入。

#### 降级条目规则

- 标题前添加 `⚠️ [需手动检查]`
- 保留原始标题、作者、年份、期刊信息
- 在附加字段中说明：
  - 原始信息来自 AI
  - DOI 校验失败
  - 需要手动核查并补全元数据

#### 好处

- 所有文献都进入 Zotero，不会静默遗漏。
- 后续只需搜索 `⚠️` 即可定位问题条目。
- 工作流闭环，风险可控。

## 7. 数据结构设计

### 7.1 内部处理中间结构

| 字段 | 说明 |
|---|---|
| `order` | 条目顺序 |
| `title` | 文献标题 |
| `author` | 第一作者姓氏 |
| `year` | 年份 |
| `journal` | 期刊名 |
| `doi` | AI 原始 DOI |
| `corrected_doi` | Crossref 校验后的 DOI |
| `needs_review` | 是否需人工检查 |

### 7.2 正常导入条目要求

正常导入的条目应至少具备以下信息：

- 类型：期刊文章
- 标题：标准标题
- 作者：至少第一作者
- 年份
- 期刊名
- DOI

### 7.3 降级导入条目要求

降级导入的条目应至少具备以下信息：

- 类型：期刊文章
- 标题：`⚠️ [需手动检查]` 加原始标题
- 作者：AI 给出的第一作者
- 年份
- 期刊名
- 附加说明：提示该条目需手动校验

## 8. 异常处理规范

### 8.1 文件相关异常

| 场景 | 处理方式 |
|---|---|
| `reference.bib` 不存在 | 红色报错并退出 |
| `.bib` 文件为空 | 黄色警告并退出 |
| `.bib` 解析失败 | 红色报错并退出 |

### 8.2 Crossref 相关异常

| 场景 | 处理方式 |
|---|---|
| DOI 校验返回无结果 | 进入模糊搜索 |
| 模糊搜索无结果 | 标记 `needs_review` |
| 返回分数低于阈值 | 标记 `needs_review` |
| 请求超时 | 记录警告并降级导入 |
| 请求被限流 | 等待后重试，最多三次 |

### 8.3 Zotero 相关异常

| 场景 | 处理方式 |
|---|---|
| API Key 无权限 | 红色报错并退出 |
| Collection 创建失败 | 红色报错并退出 |
| 单条条目导入失败 | 记录警告，继续处理剩余条目 |
| 批量导入部分失败 | 在汇总报告中列出失败项 |

### 8.4 控制台输出规范

建议使用三类状态输出：

- 成功：绿色
- 警告：黄色
- 致命错误：红色

同时建议在输出中标明当前处理进度，例如：

- 第几篇 / 共几篇
- 当前标题
- 当前状态
- 是否进入人工检查

## 9. 控制台输出设计

### 9.1 处理阶段输出

建议分三个阶段输出：

1. 读取输入文件
2. 校验与纠偏
3. 导入 Zotero

### 9.2 汇总报告字段

导入结束后，至少输出以下统计信息：

| 指标 | 说明 |
|---|---|
| 总文献数 | 从 `.bib` 中读取的总条目数 |
| 正常导入数 | 成功匹配并导入的数量 |
| 降级导入数 | 需人工检查但已导入的数量 |
| 完全失败数 | 最终未成功写入 Zotero 的数量 |

### 9.3 用户提示

脚本结束后应明确提醒用户：

- 在 Zotero 中打开本次新建的 Collection。
- 搜索 `⚠️` 快速定位问题条目。
- 手动核查这些条目的元数据后再进入论文定稿阶段。

## 11. 使用流程

### Step 1

论文写作完成后，从 AI 工具导出 `reference.bib` 文件。

### Step 2

确认 `.bib` 中条目顺序与正文引用顺序一致。

### Step 3

将 `reference.bib` 放入脚本目录。

### Step 4

配置以下参数：

- Zotero Library ID
- Zotero API Key
- Crossref 邮箱
- 输入文件路径

### Step 5

运行脚本。

### Step 6

在 Zotero 中打开新建的 `Paper_Refs_YYYYMMDD` Collection。

### Step 7

搜索 `⚠️`，检查并手动补全异常条目。

### Step 8

完成人工修正后，再进入 Word 论文引用插入阶段。

## 13. 结论

该方案的核心不是“完全自动替代人”，而是将原本最耗时、最容易出错的三个环节自动化：

- 结构化读取 AI 给出的参考文献
- 自动校验和补全 DOI
- 自动导入 Zotero 并显式标记异常条目

这样可以显著减少手动查找、逐篇导入和漏文献的风险，同时保持最终人工把关的可控性。

在单机毕业论文写作场景下，这是一个实现成本低、收益高、可快速落地的最小可用方案。
